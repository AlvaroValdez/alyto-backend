Guía de Implementación: Retención de Profit

Fecha: 16 de Enero, 2026

Resumen Ejecutivo

Implementamos con éxito un Flujo de Pago en Dos Pasos para habilitar la retención de ganancia. Esto reemplaza el flujo anterior de un solo paso.

  • Problema Anterior: Cliente paga X = Beneficiario recibe X. Ganancia = 0.
  • Solución: Cliente paga X (Precio Venta) → Beneficiario recibe Y (Precio Costo). Ganancia = X - Y (se queda en Wallet).

Componentes Clave Implementados

1. withdrawals.js (Paso 1)

  • Calcula adjustedAmount (Precio Costo) basado en tasas de Vita y Alyto.
  • Crea Transaction en BD con deferredWithdrawalPayload.
  • No ejecuta el retiro inmediatamente.

2. paymentOrders.js (Paso 2)

  • Crea Orden de Pago por el monto total (Precio Venta).
  • Idempotencia: Reutiliza orden existente si hay reintentos.
  • Maneja la limitación de que este endpoint no soporta url_notify.

3. Mecanismo de Respaldo (check-status)

Solución Robusta: Implementamos un backup crítico.

Dado que Vita no siempre envía IPN para órdenes de pago (si falta config global), agregamos el endpoint /check-status.

El Frontend llama a este endpoint automáticamente en la pantalla de éxito, forzando la ejecución del retiro si el dinero ya ingresó.

Archivos Modificados

  • src/routes/withdrawals.js: Lógica de cálculo y creación diferida.
  • src/routes/paymentOrders.js: Orquestación de pago y fallback.
  • src/routes/ipn.js: Manejo de notificaciones asíncronas.
  • src/pages/PaymentSuccess.jsx: Activación proactiva del retiro.
